Em 1967, Roland Barthes declarou a morte do autor. Em 2024, descobrimos que ele estava apenas em coma — e que agora fomos todos nós que morremos junto.

I. O FANTASMA NA MÁQUINA

Quando perguntamos a um modelo de linguagem "quem escreveu isso?", a resposta correta não é "GPT-4" ou "Claude". A resposta correta é: ninguém. E isso não é um bug — é a feature mais radical da tecnologia contemporânea.

Barthes argumentava que o texto é "um espaço de dimensões múltiplas onde se casam e se contestam escrituras variadas, nenhuma das quais é original". Se isso era verdade para a literatura do século XX, o que dizer de um sistema treinado em bilhões de textos, incapaz de distinguir suas "influências"?

O autor é uma personagem moderna, produto da nossa sociedade... é a ideologia capitalista que deu a maior importância à 'pessoa' do autor.
— Roland Barthes, "A Morte do Autor" (1967)

Mas se o autor foi uma invenção capitalista, o que fazemos quando o capitalismo inventa máquinas que escrevem sem autores? Quando a produção textual se torna tão barata que a autoria perde seu valor de mercado?

II. CÓDIGO COMO ONTOLOGIA

Considere este fragmento simples de Python que gera texto:

import random

def gerar_autor(corpus, temperatura=0.7):
    """
    Função que simula autoria através de
    amostragem probabilística de um corpus.
    """
    tokens = corpus.split()
    resultado = []
    
    for _ in range(100):
        # Quem é o autor aqui?
        # O programador? O corpus? O random.choice?
        token = random.choice(tokens)
        resultado.append(token)
    
    return " ".join(resultado)

# A autoria está distribuída em:
# 1. Quem escreveu esta função
# 2. Quem compilou o corpus
# 3. Quem escreveu os textos originais
# 4. O próprio algoritmo de randomização
# 5. Ninguém e todos simultaneamente

Esta função trivial já põe em questão toda a ontologia da autoria. Quando escalamos isso para transformers com 175 bilhões de parâmetros, a questão não é "quem é o autor?", mas sim: a categoria 'autor' ainda faz algum sentido?

A Equação da Autoria Distribuída

Se pudéssemos formalizar matematicamente a autoria em sistemas generativos, seria algo como:

A(texto) = ∑i=1n wi · Ci + ε

Onde A é autoria, Ci são os fragmentos do corpus, wi são pesos de atenção, e ε é o ruído estocástico irredutível

O problema? ε não é um bug — é onde a "criatividade" emerge. E ninguém pode reivindicá-la como propriedade.

III. IMPLICAÇÕES PRÁTICAS (OU: POR QUE ISSO IMPORTA)

Esta não é apenas uma discussão teórica. Considere estas situações reais:

  • Um estudante submete um ensaio gerado por IA. Quem plagioou quem?
  • Uma IA cria um romance best-seller. Quem recebe os royalties?
  • Um sistema gera código que viola uma patente. Quem é responsável?
  • Um modelo reproduz vieses racistas do corpus. Quem responde?

Todas essas perguntas pressupõem que exista um quem — um sujeito responsável, um autor. Mas se Barthes estava certo, se a morte do autor era inevitável, então nossa obsessão com encontrar culpados é apenas nostalgia por um conceito que já estava moribundo.

Nascimento do leitor tem de pagar-se com a morte do autor.
— Roland Barthes, novamente

Mas e se agora nem o leitor sobrevive? Quando IAs leem IAs que escrevem para outras IAs, onde está o humano nessa cadeia?

IV. CONCLUSÃO (OU: O QUE FAZER AGORA)

A resposta não é romantizar o autor humano ou demonizar a máquina. A resposta é aceitar que estamos em território novo, onde conceitos como "originalidade", "autoria" e "criatividade" precisam ser radicalmente repensados.

Talvez o que precisamos não seja proteger o autor, mas abolir de vez a fantasia de que textos têm donos. Talvez precisemos de novos vocabulários, novos frameworks legais, novas éticas que não dependam de encontrar um sujeito responsável individual.

Porque no fim, a pergunta não é "quem escreveu isso?"

A pergunta é: por que ainda precisamos perguntar?
